# Acquisition Project README

Author: Usim E. Orji
Date: July 2026

Project: Credit Card Acquisition — Funnel Analysis & Commercial Impact

Overview

This document describes the acquisition funnel analysis performed on sample event tables (Product Visits and Application Starts). It explains the business problem, datasets used, data quality findings, analysis performed, key results based on conservative assumptions, and how to reproduce the analysis and generate an executive PowerPoint.

Assumptions used in the sample analysis
- Average revenue per approved application (LTV): £200
- Approval conversion from Start → Approval: 60%
- Scenario uplift modelled: +5 percentage points (absolute) to Visits → Starts

Business problem
We have multiple acquisition channels and campaigns that generate visits to product pages, but we do not currently have a complete, channel-attributed view of how those visits convert into submitted and approved credit-card customers and revenue. Leadership requires an end-to-end funnel, commercial impact estimates (lost revenue, ROI), and prioritized recommendations to increase profitable acquisitions.

Datasets (sample provided)
- Product Visits (fields): customer_id, visit_date, channel, product, device_type, campaign_id, entry_source, visit_count
- Application Starts (fields): customer_id, application_id, start_date, product, channel, entry_source, pre_approved_flag, application_status

Key data-quality findings (sample)
- Starts table lacks an explicit campaign_id column. Starts were attributed to campaigns by joining to the most recent visit for the same customer_id.
- Small sample sizes create low-confidence high-conversion observations (e.g., C021 had 1 visit and 1 start).
- Missing datasets required for full commercial analysis: Submissions, Decisions (Approvals/Declines with revenue), and Marketing Spend by channel/campaign.

Analysis performed
1. Attribution: mapped each application start to the most recent visit for the same customer_id to attribute channel and campaign.
2. Funnel aggregation: computed Visits → Starts counts and conversion rates by channel, product, and campaign.
3. Lost revenue estimation: assumed approval conversion from Start and an average revenue per approved application to estimate revenue lost through abandonments.
4. Uplift simulation: modelled a +5pp absolute uplift to Visits → Starts and computed incremental starts, approvals and revenue.
5. Generated CSV outputs and chart PNGs and prepared an executive PowerPoint (see scripts to re-run and reproduce).

Key results (sample, conservative assumptions applied)
- Total visits: 12; total starts: 3; total abandoned visits: 9
- Visits→Starts conversion (sample): App 25.0%, Public 50.0%, OLB 0.0%
- Estimated lost revenue from abandonments (9 abandoned starts * 60% approval * £200): £1,080
- +5pp uplift impact (sample): incremental revenue ≈ £72

Recommendations
1. Critical: supply Submissions, Decisions (Approved/Declined with revenue/LTV), and Marketing Spend by channel/campaign for the same timeframe — required to compute CAC and ROI.
2. Prioritise App UX optimisation (highest absolute visits); run Start→Submission A/B tests to reduce abandonment.
3. Validate high-conversion small channels (C021/Public) at scale before reallocating budget.
4. Deduplicate events by application_id and ensure session_id + timestamps are available for robust attribution.

How to reproduce
- Install dependencies: pip install pandas numpy matplotlib seaborn python-pptx openpyxl
- Run the analysis (example):
  python scripts/acquisition_analysis_with_scenarios.py
- Generate the PowerPoint (after the analysis outputs are created):
  python scripts/make_presentation.py

Files added

- scripts/make_presentation.py — programmatic PPTX generation using analysis outputs

Notes
